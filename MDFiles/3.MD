# 🎙️ Edge-AI Speech Disentanglement — Locked Engineering Plan v3

This document supersedes `1.MD` and `2.MD`. It integrates (a) the architectural fixes from v2 and (b) the six literature-validated additions + two corrections from the April 2026 research pass. This is the version to implement from.

Primary sources backing every non-obvious choice are cited at the bottom (`§14`).

---

## 1. System Definition

**Task.** Given a continuous 16 kHz audio stream, emit a detection event iff (a) a user-defined trigger word is present **and** (b) it was spoken by the enrolled user.

**Operating envelope.** SNR ∈ [-5, 30] dB · distance ∈ [0.5, 5] m · male + female · multi-accent · xRT < 0.2 s · params < 3 M · FA < 1/hr.

**Non-goals (explicit).** Full-vocabulary ASR, multi-user simultaneous detection, diarization, on-device speaker enrollment GUI.

---

## 2. Final Model Architecture

### 2.1 Shared Frontend (GPU, differentiable)
```
raw_wav(16kHz) → PreEmphasis(0.97) → MelSpec(n_mels=40, win=25ms, hop=10ms)
              → LearnablePCEN(α, δ, r, smooth_coef) → LayerNorm(freq)
```
PCEN parameters (α, δ, r, smoothing coef) are **learnable per-channel**. Fixed PCEN leaves ~2% TA on the table at -5 dB SNR. The 2024 LEAF-analysis paper proves PCEN is the only part of learnable-frontend stacks that meaningfully trains — so this frontend is equivalent to LEAF in practice at a fraction of the compute.

### 2.2 Backbone — BC-ResNet-8 (modified) with TRM branching
* Channels: base = 32 → backbone ~780 K params.
* **Shared trunk:** first 4 BC-ResNet blocks.
* **Task Representation Module (TRM):** a small learnable linear projection **per task** that transforms the shared feature tensor before it enters each branch (replaces v2's PCGrad as the primary gradient-decoupling mechanism — PK-MTL showed TRM gives bigger and more reliable gains, and the 2025 GCond paper found PCGrad can *underperform* a plain baseline on some benchmarks).
  * `shared_feat → TRM_kws (1×1 conv + LN) → KWS branch`
  * `shared_feat → TRM_sv  (1×1 conv + LN) → SV branch`
  * Cost: ~15 K params.
* **Branches:** 2 BC-ResNet blocks each.
* **KWS pooling:** frequency-wise pooling → 128-d.
* **SV pooling:** **Multi-Query Multi-Head Attention (MQMHA) pooling** (4 heads, 2 queries) — documented ~8% relative EER improvement over mean+std attentive pooling on VoxCeleb at comparable param count. Output: 256-d (mean‖std) internal → projected to 192-d.

### 2.3 Two Heads + FiLM Conditioning
| Head | Input | Layer | Output |
|------|-------|-------|--------|
| KWS  | 128-d pooled, **FiLM-modulated by SV embedding** | Linear → Dropout(0.1) → Linear | V+1 logits (vocab + "unknown") |
| SV   | 256-d MQMHA pooled  | Linear → BN → Linear | 192-d L2-normalized embedding |

**FiLM (speaker-conditioned KWS) — the key FA<1/hr structural lever:**
```
sv_emb (192-d, detached gradient) → MLP(2 layers, ~8K params)
                                  → γ (128), β (128)
kws_feat' = γ ⊙ kws_feat + β
```
The KWS classifier's effective weights depend on *who* is speaking. Imposter audio flows through the wrong (γ, β) and the KWS logit collapses — structural rejection, not threshold-based. The Google 2023 paper (arXiv 2311.03419) reports 2–6% relative EER reduction at <1% param overhead — our highest-leverage single addition for the FA<1/hr gate.

SV embedding is fed into the FiLM generator **with stop-gradient on the forward path** to prevent the KWS loss from corrupting SV clustering. FiLM generator weights train normally.

SV dim is **192** to match ECAPA's native dim (removes the silent projection mismatch v1 had).

### 2.4 Parameter Budget
| Component | Params |
|-----------|--------|
| Frontend (PCEN learnable) | ~0.2 K |
| BC-ResNet-8 shared trunk (4 blocks) | ~420 K |
| TRM_kws + TRM_sv | ~15 K |
| KWS branch (2 blocks) | ~180 K |
| SV branch (2 blocks) | ~180 K |
| MQMHA pooling | ~30 K |
| KWS head + FiLM generator | ~25 K |
| SV head (Linear+BN+Linear to 192-d) | ~55 K |
| ECAPA distillation projection heads (3×) | ~15 K |
| WavLM hint-layer projection (1×1) | ~10 K |
| **Student total** | **~1.13 M** |

Well inside the 3 M cap, ~60% headroom for QAT scale/zero-point tensors and any late-stage capacity additions.

## 3. Teacher Design & Distillation

Neither WavLM nor ECAPA emits what the student needs directly. The repair is explicit below.

### 3.1 Teacher A — WavLM-Base+ (phonetic, noise-robust)
* Freeze the 12-layer transformer.
* **KWS probing head** (2-layer MLP, ~50 K params) trained once on Speech Commands + LibriPhrase, reading WavLM layer-9 features. This head is the logit source for KD. One-time cost ~2 Kaggle GPU-hours.
* **Logit KD:** `KL(student_kws_logits / T ‖ probed_teacher_logits / T)`, T ∈ [2, 4].
* **Layer-ensemble hint-layer MSE (SKILL-style):** rather than distilling student features against a single WavLM layer, match against a **learnable weighted sum of WavLM layers {4, 8, 11}** — hierarchical clustering analyses (SKILL, Interspeech 2024) show these are the cluster representatives and beat any single-layer choice. Weights `w_4, w_8, w_11` are 3 learnable scalars, softmax-normalized per step.
  ```
  teacher_hint = softmax(w_4, w_8, w_11) · [WavLM_4, WavLM_8, WavLM_11]
  L_hint       = MSE(proj_1x1(student_feat_post_trunk), teacher_hint)
  ```

### 3.2 Teacher B — ECAPA-TDNN (biometric), multi-level
* Freeze `speechbrain/spkrec-ecapa-voxceleb` (192-d output).
* **Multi-level distillation** (replaces v2's final-embedding-only cosine): follows "Distilling Multi-Level X-vector Knowledge" (arXiv 2303.01125) which reports ~5% EER improvement over single-level.
  * `L_ecapa_final = 1 - cos(student_emb, ecapa_final_emb)`
  * `L_ecapa_b2    = MSE(proj_b2(student_feat), ecapa_block2)`   — 1×1 conv projection ~5 K
  * `L_ecapa_b3    = MSE(proj_b3(student_feat), ecapa_block3)`   — 1×1 conv projection ~5 K
  * `L_sv_kd = L_ecapa_final + 0.3 · (L_ecapa_b2 + L_ecapa_b3)`
* **Pairing (corrected from v2):** teacher sees the **exact same augmented audio as the student**. The asymmetric "student augmented, teacher clean" scheme causes student collapse to teacher's clean manifold (arXiv 2312.12153, correlation-KD). ECAPA was pretrained on augmented VoxCeleb2 and is already noise-robust — leverage that.

### 3.3 Losses

**Speaker Verification loss — Sub-center AAM-Softmax (K=3 centers):**
Sub-center AAM handles intra-class variance (and noisy VoxCeleb labels) far better than single-center AAM. The change vs. plain AAM is a reshape of the weight matrix — no extra params, one-line code change.
```
L_sv = SubCenterAAMSoftmax(student_emb, y_spk, K=3, m=0.2, s=30)
```

**Keyword Spotting loss — Focal Loss with phonetic hard-negative mining** (see §4.2):
```
L_kws = Focal(student_kws_logits, y_word, γ=2.0, α_pos=0.25)
```

**Total multi-task objective:**
```
L_total =
    λ_kws       · L_kws
  + λ_sv        · L_sv
  + λ_kd_logit  · T² · KL(student_kws / T ‖ wavlm_probe / T)
  + λ_kd_hint   · L_hint                     # layer-ensemble WavLM KD
  + λ_kd_ecapa  · L_sv_kd                    # multi-level ECAPA KD
```

**Gradient decoupling:** TRM is the primary mechanism (see §2.2). **PCGrad is enabled as a secondary guard only after epoch 5** — we keep it behind a flag because evidence is mixed (PCGrad helps in some settings, underperforms in others). Default: off. Turn on if training diverges.

## 4. Data Pipeline

### 4.1 Sources
Speech Commands v0.02 · VoxCeleb1 · LibriPhrase · MUSAN · OpenSLR RIRs · Bark / XTTS-v2 for custom trigger synthesis.

### 4.2 Hard-Negative Confusable Mining (the FA<1/hr lever, reinforcing FiLM)
1. G2P the trigger word via `g2p-en` → ARPAbet phones.
2. For every word in Speech Commands + LibriPhrase + CMUdict, compute weighted Levenshtein on phones with a TIMIT-derived phone-confusion matrix.
3. Top-K (K=50) nearest words become **mined negatives**, sampled at 3× the rate of random negatives during training.
4. For the TTS-generated trigger, synthesize the top-K confusables in the same voices — gives speaker-matched negatives and stress-tests the FiLM conditioning.

### 4.3 TTS Distribution Gap (fix, not band-aid)
* Every TTS clip passes through the **same RIR + MUSAN pipeline** as real audio before training.
* Batch composition: real:TTS = 3:1. TTS never exceeds 25% of any batch.
* Validation set contains **zero TTS** — only real mic recordings.

### 4.4 SNR Curriculum
| Epoch range | SNR sampling |
|-------------|--------------|
| 0–5   | U[20, 30] dB |
| 5–15  | U[5, 30] dB |
| 15–30 | U[-5, 30] dB |
| 30+   | U[-5, 30] dB with 20% chance of pure noise (labelled "unknown") |

### 4.5 On-the-Fly Augmentation Order
```
wav → speed_perturb(0.9–1.1, p=0.3)
    → RIR convolve (p=0.7, sampled by target distance bin)
    → MUSAN mix at curriculum SNR
    → (optional) codec sim: µ-law / AMR-NB (p=0.2)
    → MelSpec → PCEN → SpecAugment (2 freq × 2 time masks)
```
Teacher (ECAPA) sees the **same augmented waveform** as the student (see §3.2).

## 5. Training Schedule (4 stages, ~36 Kaggle GPU-hrs)

| Stage | Data | Loss terms active | Epochs | Purpose |
|-------|------|-------------------|--------|---------|
| 0. WavLM probe pretraining | SC + LibriPhrase | CE on probing head only | ~2 hr | One-time: train WavLM → KWS probe |
| 1. Backbone warm-up | SC only | Focal only | 10 | Stabilize PCEN + shared trunk |
| 2. Joint MT + all KD | SC + VoxCeleb1 + LibriPhrase | All 5 terms, FiLM active, TRM active | 25 | Full multi-task distillation |
| 3. Trigger fine-tune + online τ_s | LibriPhrase + TTS custom trigger | Focal + Sub-center AAM + cos-KD + τ_s soft penalty | 8 | Specialize + FAR-aware embedding |
| 4. QAT | Same as Stage 3 | Same + fake-quant | 4 | INT8-aware weights |

**Stage 1 is non-negotiable** — skipping it makes PCEN learnable params diverge under multi-task gradients (observed failure, not theoretical).

## 6. Enrollment, Verification & Online τ_s Calibration

### 6.1 Offline enrollment (one-time per user)
1. User records **5 clean utterances** of the trigger word.
2. For each: extract SV embedding, L2-normalize.
3. User template = mean of the 5 embeddings, re-normalized. 192 × float32 = 768 bytes stored.
4. Compute per-user τ_k, τ_s from enrollment + 2 min of the user's own negative speech.

### 6.2 Runtime decision
```
kws_prob = softmax(kws_logits)[trigger]
sv_sim   = cos(sv_emb, template)
detect   = (kws_prob > τ_k) AND (sv_sim > τ_s)
```

### 6.3 Online τ_s calibration during Stage 3 (correction to v2)
v2 treated τ_s as a pure deployment knob. PK-MTL evidence shows training-time FAR-awareness is needed — without it the embedding space isn't clustered tightly enough for any τ_s to hit <0.15% FAR.

**Implementation:**
* Every 500 training steps, run a forward pass on a held-out imposter pool (~2000 clips).
* Compute the τ_s that yields the target FA/hr on this pool.
* Compute the FRR at that τ_s on a held-out positive pool.
* Add `λ_far_aware · FRR_at_target_far` as a soft penalty to the training loss for the next 500 steps.
* This pushes the embedding space to be separable at the deployment operating point, not just in aggregate.

## 7. Evaluation Protocol

### 7.1 Test matrices
* **Speakers:** 20 held-out (10 M, 10 F, multi-accent), never seen in training.
* **Noise types:** crowd, babble, traffic (MUSAN held-out) × SNR {-5, 0, 5, 10, 20} dB.
* **Distance bins:** {0.5, 1, 2, 3, 5} m via held-out RIR set.
* **Clean set:** studio mic, no RIR, no noise.

### 7.2 Metric definitions
* **TA-clean / TA-noisy:** recall on positive trigger utterances from the enrolled speaker.
* **FA/hr:** 1 hr of continuous distractor audio (mix of: other speakers saying the trigger, enrolled speaker saying confusables, pure noise). Count detections / 1 hr.
* **xRT:** single-sample, batch=1, streaming mode, on target hardware (§8). Not Kaggle GPU.

### 7.3 Ablation table (required in final report)
| Ablation                                                   | TA-noisy Δ | FA/hr Δ |
|------------------------------------------------------------|------------|---------|
| –PCEN (log-Mel only)                                       |            |         |
| –WavLM KD (all layers)                                     |            |         |
| –Layer-ensemble WavLM KD (single layer 9 instead of {4,8,11}) |         |         |
| –ECAPA multi-level KD (final embedding only)               |            |         |
| –FiLM speaker conditioning                                 |            |         |
| –Sub-center AAM (K=1)                                      |            |         |
| –MQMHA pooling (mean+std only)                             |            |         |
| –TRM (shared trunk direct → branches)                      |            |         |
| –Online τ_s calibration                                    |            |         |
| –Hard-negative confusable mining                           |            |         |
| –SNR curriculum (flat -5 to 30 from epoch 0)               |            |         |

## 8. Edge Deployment (the xRT<0.2s gate)

**Target hardware:** Raspberry Pi 4B (Cortex-A72) as conservative spec. Re-benchmark if deliverable hardware differs.

### 8.1 Export path
```
PyTorch (QAT, fake-quant) → ONNX (opset 17) → ORT INT8 static quantization
→ benchmark via onnxruntime with 1 thread
```

### 8.2 Streaming inference
* Chunked input: 160 ms (1600 samples), 40 ms overlap.
* **PCEN smoothing state** is maintained across chunks — export as a stateful op or manage state in host code. This is the non-obvious streaming cost.
* **BC-ResNet conv state** maintained via ring-buffer (reference: Google `kws_streaming` framework).

### 8.3 Latency budget (Pi 4B, INT8, single thread)
| Stage | Time (ms) |
|-------|-----------|
| MelSpec + PCEN | ~8 |
| BC-ResNet INT8 | ~35 |
| FiLM modulation | ~1 |
| Both heads | ~3 |
| MQMHA pooling | ~2 |
| Cosine + thresh | ~1 |
| **Total / chunk** | **~50** |

xRT = 50/160 ≈ 0.31 — **over budget on Pi 4B**. Mitigations in order of preference:
1. Reduce BC-ResNet base channels 32 → 24 (saves ~35% compute, expected ~2% TA drop).
2. Move to 200 ms chunks (xRT = 50/200 = 0.25, still over).
3. Fall back to Cortex-A76-class silicon (Pi 5, modern Android).
Decision deadline: end of Stage 1.

## 9. Hyperparameter Search (Optuna)

**Scoped to Stage 2** — end-to-end Optuna on a 36-hour pipeline is infeasible on Kaggle.

* Sampler: TPE, 40 trials, median pruner.
* Search space:
  * λ_kws, λ_sv, λ_kd_logit, λ_kd_hint, λ_kd_ecapa: log-U[0.1, 10]
  * T ∈ [2, 5]
  * Sub-center AAM margin ∈ [0.15, 0.35], scale ∈ {20, 30, 40}
  * LR ∈ log-U[5e-5, 5e-3]
* **Objective:** `TA_noisy − 10 · max(0, FA_per_hr − 1)`. Linear penalty (heavy penalties collapse TPE's acquisition surface).

## 10. Kaggle Execution

* All datasets mounted via `/kaggle/input/`. Pre-upload VoxCeleb1 as a Kaggle Dataset — `torchaudio.datasets.VOXCELEB1identification` download path is frequently broken.
* Checkpoint every epoch **and** every 90 minutes (wall-clock), keyed by Optuna trial ID.
* On resume: reload `model + optimizer + scheduler + Optuna study DB + RNG state`. Omitting RNG state silently changes augmentation seed → non-reproducible resumption.
* `torch.compile(model, mode="reduce-overhead")` after Stage 1 — ~15% train speedup on T4/P100.

## 11. Risk Register

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| WavLM probing head underfits | M | M | Layer-ensemble KD ({4,8,11}) already mitigates single-layer bias |
| TRM underpowered vs. PCGrad for conflict | L | M | PCGrad fallback behind flag, enable from epoch 5 if divergence |
| FiLM conditioning over-specializes on enrollment speakers | M | M | Train FiLM generator with wide speaker distribution (VoxCeleb1 full); freeze at deployment |
| TTS voices collapse to a few clusters | H | H | XTTS-v2 with 100+ real reference voices; cluster-analysis check before Stage 2 |
| INT8 quantization drops TA > 3% | M | H | QAT (Stage 4), not PTQ |
| FA/hr budget violated by confusables | M | H | Online τ_s calibration + adversarial confusable set + FiLM gating |
| Sub-center AAM causes mode collapse (1 of K centers unused) | L | L | Log center-usage histogram each epoch; prune unused centers |
| VoxCeleb1 download blocked | H | L | Pre-upload as Kaggle Dataset |
| Multi-level ECAPA projection heads overfit | L | L | Weight decay 1e-4 on projection heads only |

## 13. Complete Changelog (v1 → v3)

### v2 deltas (carried forward)
1. WavLM KD now uses a trained probe + hint-layer MSE (not raw logit KL on WavLM outputs).
2. SV embedding dim set to 192 to match ECAPA (removes silent projection mismatch).
3. Backbone trunk partially shared (split at block 4).
4. SNR curriculum replaces flat -5/30 dB sampling.
5. TTS data routed through the real-audio augmentation pipeline before training.
6. Hard-negative confusable mining is explicit (G2P + phone-weighted Levenshtein + TIMIT confusion matrix).
7. Enrollment + per-user thresholds added.
8. Edge deployment path with concrete Pi 4B latency budget.
9. Optuna scoped to Stage 2; objective reshaped to linear penalty.
10. PCGrad was the primary MTL mechanism.

### v3 deltas (new in this document)
11. **Multi-layer ECAPA distillation** — block-2 + block-3 feature MSE alongside final-embedding cosine (arXiv 2303.01125).
12. **Sub-center AAM-Softmax (K=3)** replaces plain AAM-Softmax (PMS-Conformer, 2024).
13. **TRM (Task Representation Module) replaces PCGrad** as the primary MTL gradient-decoupling method; PCGrad kept as optional epoch-5+ fallback (PK-MTL + GCond evidence).
14. **FiLM speaker-conditioning** from SV embedding into KWS branch feature map (arXiv 2311.03419). The single highest-leverage change for FA<1/hr.
15. **Layer-ensemble WavLM KD** with 3 learnable softmax weights on layers {4, 8, 11} (SKILL, Interspeech 2024).
16. **MQMHA pooling** replaces mean+std attentive statistics pooling on the SV head (arXiv 2110.05042).
17. **Symmetric teacher/student augmentation** in ECAPA distillation (correlation-KD, arXiv 2312.12153).
18. **Online τ_s calibration** during Stage 3 with FAR-aware soft penalty — τ_s becomes a training signal, not purely a deployment knob.
