# 🎙️ Edge-AI Speech Disentanglement System

## 1. Project Overview & Target KPIs
This project implements a multi-task Knowledge Distillation pipeline to create a highly robust, extreme-edge AI model for joint Keyword Spotting (KWS) and Speaker Verification (SV). The system operates under severe noise (-5dB to 30dB SNR) and distance variations (0.5m to 5m).

### Definitive Target Benchmarks
* **True Acceptance (Clean):** $\ge$ 99%
* **True Acceptance (Noisy):** $\ge$ 90%
* **False Acceptance Rate (FA):** < 1/hr
* **Execution Time (xRT):** < 0.2 seconds
* **Model Parameter Limit:** < 3M

---

## 2. Kaggle Data Architecture & Pipeline
Due to Kaggle's 73GB `/kaggle/working/` disk limit, data must be fetched dynamically and audio features must be extracted on the GPU. **Do not pre-compute and save Mel-spectrograms to disk.**

### 2.1 Data Sources & Ingestion
* **Keyword Spotting Data (Google Speech Commands v0.02):**
    * *Usage:* Target words and hard-negative mining.
    * *Fetch:* `torchaudio.datasets.SPEECHCOMMANDS(download=True)`
* **Speaker Verification Data (VoxCeleb 1):**
    * *Usage:* Imposter rejection and SV metric learning.
    * *Fetch:* `torchaudio.datasets.VOXCELEB1identification(download=True)`
* **Joint Phrase/Speaker Data (LibriPhrase):**
    * *Usage:* Multi-task fine-tuning.
    * *Fetch:* Hugging Face `datasets.load_dataset("LibriPhrase/LibriPhrase")`
* **Noise Profiles (MUSAN):**
    * *Usage:* Injecting traffic, babble, and crowd noise (-5dB to 30dB).
    * *Fetch:* `!wget https://www.openslr.org/resources/17/musan.tar.gz`
* **Distance Simulation (OpenSLR RIRs):**
    * *Usage:* Applying Room Impulse Responses to simulate 0.5m–5m distance.
    * *Fetch:* `!wget https://www.openslr.org/resources/28/rirs_noises.zip`
* **Custom Target Word (Zero-Shot TTS):**
    * *Usage:* Generating the specific custom trigger word in 100+ voices using Hugging Face's `suno/bark` or `coqui/XTTS-v2`.

### 2.2 On-the-Fly Data Augmentation (GPU Dataloader)
1.  Read 1D raw `.wav` file into memory.
2.  Randomly select an RIR and convolve it with the clean audio (distance simulation).
3.  Randomly select a MUSAN noise file and mix it at a random SNR between -5dB and 30dB.
4.  Apply SpecAugment (masking random time and frequency bands) during training.

## 3. Feature Engineering: PCEN Frontend
Standard log-Mel spectrograms fail when noise is louder than the signal (-5dB). The first layer of the PyTorch neural network must be a dynamic Per-Channel Energy Normalization (PCEN) layer executed on the GPU.

PCEN performs automatic gain control, squashing stationary noises (like traffic drones) and enhancing transient speech spikes.
* **Transforms Pipeline:** `torchaudio.transforms.MelSpectrogram` $\rightarrow$ Custom PCEN Module.

## 4. Model Architectures & Distillation Pipeline

### 4.1 The Teacher Ensemble (Unconstrained)
Instead of training a massive teacher from scratch and burning Kaggle GPU hours, use two pre-trained SOTA models to supervise the student.

* **Teacher 1 (Speaker Verification): ECAPA-TDNN**
    * *Role:* Extracts pure biometric speaker embeddings, ignoring phonetic content.
    * *Source:* `speechbrain/spkrec-ecapa-voxceleb`
    * *Validation Metric:* Must achieve Equal Error Rate (EER) < 1.0% offline.
* **Teacher 2 (Keyword/Noise Robustness): WavLM Base+**
    * *Role:* Provides highly accurate, noise-resistant phonetic probability distributions.
    * *Source:* `microsoft/wavlm-base-plus` (pretrained with masked speech denoising).
    * *Validation Metric:* Must achieve Noisy TA > 98%.

### 4.2 The Student Target Model (< 3M Params)
* **Architecture:** BC-ResNet (Broadcasting Residual Network).
* **Why BC-ResNet?:** It utilizes 1D frequency and time processing paths and broadcasts them together. It is heavily documented as the SOTA architecture for extreme-noise keyword spotting under 1M parameters, leaving plenty of parameter budget for the SV projection head.
* **Output Structure:** Two parallel linear layers appended to the final pooling layer.
    * *Head A:* KWS Logits (Probability distribution over vocabulary + "Unknown").
    * *Head B:* SV Embedding (128-dimensional or 256-dimensional dense vector).

## 5. Multi-Task & Contrastive Learning Objectives

The network is optimized using a weighted multi-task loss function to balance word detection, speaker identity, and teacher knowledge transfer:

$$L_{total} = \lambda_1 L_{KWS} + \lambda_2 L_{SV} + \lambda_3 L_{KD}$$

### 5.1 Keyword Spotting Loss ($L_{KWS}$)
* **Algorithm:** Focal Loss.
* **Strategy (Hard Negative Mining):** Standard Cross-Entropy will fail the FA < 1/hr metric. Focal Loss heavily penalizes the network for being "unsure" about similar words. The dataloader must intentionally feed the model rhyming words spoken by the correct speaker to force a sharp phonetic decision boundary.

### 5.2 Speaker Verification Loss ($L_{SV}$)
* **Algorithm:** ArcFace (Additive Angular Margin Loss).
* **Strategy:** ArcFace forces the model to map all audio from the target speaker into a tight geometric cluster in the latent space while mathematically enforcing a massive angular distance penalty against all imposter audio from VoxCeleb.

### 5.3 Knowledge Distillation Loss ($L_{KD}$)
* **Algorithm:** Kullback-Leibler (KL) Divergence.
* **Strategy:** Calculate the KL Divergence between the Student's KWS logits and the WavLM Teacher's soft-label probabilities. Apply a Temperature parameter ($T$) to soften the distribution, allowing the student to learn intra-class phonetic similarities.

---

## 6. Hyperparameter Tuning Strategy

Manual tuning of the loss weights and margins is impossible. An automated **Optuna Study** will be executed in the Kaggle notebook.

* **Algorithm:** Tree-structured Parzen Estimator (TPE).
* **Search Space:**
    * Loss Weights: $\lambda_1, \lambda_2, \lambda_3$ (Range: 0.1 to 10.0)
    * Distillation Temperature: $T$ (Range: 1.0 to 5.0)
    * ArcFace Margin parameter (Range: 0.1 to 0.5)
    * Optimizer: AdamW Learning Rate & Weight Decay
* **Objective Function:**
    * Maximize: `Noisy_TA_Score`
    * Heavy Penalty Factor applied if `False_Acceptance_Rate` > 0.00028 (which equates to roughly 1 FA per hour of test audio).

## 7. Kaggle Session Execution & Persistence
Kaggle automatically kills kernels after 12 hours. The training loop must be strictly engineered for checkpointing.

1.  **Mounting:** Datasets must be mounted via Kaggle's `/kaggle/input/` to avoid hitting disk limits.
2.  **Callback Logic:** At the end of every epoch, execute:
    ```python
    torch.save({
        'epoch': epoch,
        'model_state_dict': student_model.state_dict(),
        'optimizer_state_dict': optimizer.state_dict(),
        'best_metric': current_best_metric
    }, '/kaggle/working/bc_resnet_checkpoint.pth')
    ```
3.  **Initialization:** The notebook must check for the existence of `/kaggle/working/bc_resnet_checkpoint.pth` at startup and automatically load the state dictionary if found, ensuring continuous training across multiple 12-hour Kaggle sessions until convergence.
