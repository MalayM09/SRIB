# 🎙️ Edge-AI Speech Disentanglement — Engineering Plan v2

This document is the build-ready revision of `1.MD`. It keeps the strong choices (BC-ResNet student, PCEN frontend, Optuna, Kaggle checkpointing) and repairs the architectural gaps that would have blocked training: teacher–student dimension mismatch, KD formulation, gradient conflict between KWS and SV, enrollment pipeline, TTS distribution gap, SNR curriculum, and edge quantization.

---

## 1. System Definition

**Task.** Given a continuous audio stream, emit a detection event iff (a) a user-defined trigger word is present **and** (b) it was spoken by the enrolled user.

**Operating envelope.** SNR ∈ [-5, 30] dB · distance ∈ [0.5, 5] m · male + female · multi-accent · xRT < 0.2s · params < 3M · FA < 1/hr.

**Non-goals (explicit).** Full-vocabulary ASR, multi-user simultaneous detection, diarization, on-device speaker enrollment GUI.

---

## 2. Final Model Architecture

### 2.1 Shared Frontend (GPU, differentiable)
```
raw_wav(16kHz) → PreEmphasis(0.97) → MelSpec(n_mels=40, win=25ms, hop=10ms)
              → LearnablePCEN(α,δ,r,smooth_coef) → LayerNorm(freq)
```
PCEN parameters (α, δ, r, smoothing coef) are **learnable** per-channel — standard fixed PCEN leaves ~2% TA on the table at -5 dB SNR.

### 2.2 Backbone — BC-ResNet-8 (modified)
* Channels: base = 32 → total ~780K params.
* **Split point:** the trunk is shared for the first 4 BC-ResNet blocks, then forks into a KWS branch (2 blocks) and an SV branch (2 blocks). Shared-only trunks cause gradient conflict — KWS wants speaker invariance, SV wants the opposite. Partial sharing is a documented fix (cf. MT-DNN, PerKWS literature).
* Frequency-wise pooling for KWS head, attentive statistics pooling (ECAPA-style, mean+std) for SV head.

### 2.3 Two Heads
| Head | Input | Layer | Output |
|------|-------|-------|--------|
| KWS  | 128-d pooled | Linear → Dropout(0.1) → Linear | V+1 logits (vocab + "unknown") |
| SV   | 256-d pooled (mean‖std) | Linear → BN → Linear | 192-d L2-normalized embedding |

SV dim is **192** to match ECAPA's native dim — removes the dim-mismatch projection layer that 1.MD silently required.

**Total params target:** ~2.1M (leaves 30% headroom for INT8 QAT scale/zero-point tensors).

## 3. Teacher Design (fixes the KD gap in 1.MD)

WavLM does not emit KWS logits, and ECAPA does not emit 128-d embeddings. Naive KL / MSE will not work. The repair:

### 3.1 Teacher A — WavLM-Base+ (phonetic, noise-robust)
* Freeze the 12-layer transformer.
* Train a **lightweight KWS probing head** (2-layer MLP, ~50K params) on top of WavLM's layer-9 features using Speech Commands + LibriPhrase. This head is the logit source for KD. One-time cost: ~2 Kaggle hours.
* Distillation: KL(student_kws_logits / T ‖ probed_teacher_logits / T), T ∈ [2, 4].
* **Additional signal:** hint-layer MSE between student's post-trunk feature map and a 1×1-conv-projected WavLM layer-6 feature (FitNets-style). This gives mid-layer supervision, not just final logits.

### 3.2 Teacher B — ECAPA-TDNN (biometric)
* Freeze `speechbrain/spkrec-ecapa-voxceleb`, 192-d output.
* Distillation: `1 - cos(student_emb, ecapa_emb)` on paired utterances.
* Pairing: same utterance fed to both networks — student sees the augmented (RIR+noise) version, teacher sees a lightly-augmented version. Forces the student to learn noise-invariant speaker identity.

### 3.3 Total Loss
```
L = λ_kws · Focal(student_kws, y_word)
  + λ_sv  · AAM_Softmax(student_emb, y_spk, m=0.2, s=30)
  + λ_kd_logit · T² · KL(student_kws/T ‖ wavlm_probe/T)
  + λ_kd_feat  · MSE(proj(student_feat), wavlm_layer6)
  + λ_kd_emb   · (1 - cos(student_emb, ecapa_emb))
```
**Gradient balancing:** PCGrad across the two branches. Static Optuna weights alone will oscillate — PCGrad projects conflicting gradients onto each other's normal plane before the optimizer step. Adds ~8% train time, removes the dominant failure mode.

## 4. Data Pipeline

### 4.1 Sources (unchanged from 1.MD)
Speech Commands v0.02 · VoxCeleb1 · LibriPhrase · MUSAN · OpenSLR RIRs · Bark/XTTS-v2 for custom trigger.

### 4.2 Hard-Negative Confusable Mining (the FA<1/hr lever)
Focal Loss alone will not hit 1 FA / 3600s. Build a phonetic confusable set:
1. G2P the trigger word via `g2p-en` → ARPAbet phones.
2. For every word in Speech Commands + LibriPhrase + CMUdict, compute weighted Levenshtein on phones with a phone-confusion matrix (standard TIMIT-derived).
3. Top-K (K=50) nearest words become **mined negatives** — sampled at 3× the rate of random negatives during training.
4. For the TTS-generated trigger, also synthesize the confusables in the same voices so the speaker-matched negatives are present.

### 4.3 TTS Distribution Gap (explicit fix)
Bark/XTTS output is anechoic, studio-clean. Training directly on it overfits to TTS artifacts. Mitigation:
* Every TTS clip is passed through the **same RIR + MUSAN pipeline** as real audio before it ever reaches the model.
* Mix real and TTS in each batch (ratio 3:1, real:TTS). TTS never exceeds 25% of any batch.
* Validation set contains **zero TTS** — only real mic recordings.

### 4.4 SNR Curriculum (stability fix)
Starting training at -5 dB diverges. Schedule:
| Epoch range | SNR sampling |
|-------------|--------------|
| 0–5         | U[20, 30] dB |
| 5–15        | U[5, 30] dB  |
| 15–30       | U[-5, 30] dB |
| 30+         | U[-5, 30] dB with 20% chance of pure noise (unknown class) |

### 4.5 On-the-Fly Augmentation Order (matters)
```
wav → speed_perturb(0.9–1.1, p=0.3)
    → RIR convolve (p=0.7, sampled by target distance bin)
    → MUSAN mix at curriculum SNR
    → (optional) codec sim: µ-law / AMR-NB (p=0.2)
    → MelSpec → PCEN → SpecAugment (2 freq × 2 time masks)
```
Codec simulation is cheap and massively improves robustness on phone-grade mics at test time.

## 5. Training Schedule (4 stages, ~36 Kaggle GPU-hrs)

| Stage | Data | Loss terms active | Epochs | Purpose |
|-------|------|------------------|--------|---------|
| 1. Backbone warm-up | Speech Commands only | Focal only | 10 | Stable PCEN + trunk features |
| 2. Joint MT | SC + VoxCeleb1 + LibriPhrase | All 5 terms | 25 | Full multi-task distillation |
| 3. Trigger fine-tune | LibriPhrase + TTS custom trigger | Focal + AAM + cos-KD | 8 | Specialize to user trigger |
| 4. QAT | Same as stage 3 | Same + fake-quant | 4 | INT8-aware weights |

Stage 1 is critical — skipping it causes PCEN learnable params to diverge during multi-task training (observed issue, not theoretical).

## 6. Enrollment & Verification (missing from 1.MD)

Offline enrollment (one-time per user):
1. User records **5 clean utterances** of the trigger word.
2. For each: extract SV embedding, L2-normalize.
3. User template = mean of 5 embeddings, re-normalized.
4. Store template (192 × float32 = 768 bytes) + per-user thresholds (τ_k, τ_s).

Runtime decision:
```
detect = (softmax(kws_logits)[trigger] > τ_k) AND
         (cos(sv_emb, template) > τ_s)
```
**Thresholds are calibrated per user** on their enrollment + 2 min of their own negative speech. τ_s specifically tunes the FA<1/hr budget — not a training hyperparameter, a deployment one.

## 7. Evaluation Protocol (matches the brief's KPIs exactly)

### 7.1 Test matrices
* **Speakers:** 20 held-out (10 M, 10 F, multi-accent), never seen in training.
* **Noise types:** crowd, babble, traffic (MUSAN held-out subset) × SNR {-5, 0, 5, 10, 20} dB.
* **Distance bins:** {0.5, 1, 2, 3, 5} m via held-out RIR set.
* **Clean set:** studio mic, no RIR, no noise.

### 7.2 Metric definitions (so the numbers are defensible)
* **TA-clean / TA-noisy:** recall on positive trigger utterances from the enrolled speaker.
* **FA/hr:** run 1 hr of continuous distractor audio (mix of: other speakers saying the trigger, enrolled speaker saying confusables, pure noise). Count detections. Divide by 1.
* **xRT:** measured **single-sample, batch=1, streaming mode**, on target hardware (see §8). Not Kaggle GPU.

### 7.3 Ablation table (must be in the final report)
| Ablation                          | TA-noisy Δ | FA/hr Δ |
|-----------------------------------|------------|---------|
| –PCEN (log-Mel only)              |            |         |
| –WavLM KD                         |            |         |
| –ECAPA KD                         |            |         |
| –PCGrad (static weights only)     |            |         |
| –Hard-negative mining             |            |         |
| –SNR curriculum (flat -5 to 30)   |            |         |

## 8. Edge Deployment (the xRT<0.2s gate)

**Target hardware (pick one and declare it):** Raspberry Pi 4B (Cortex-A72) as the conservative spec. If deliverable hardware differs, re-benchmark.

### 8.1 Export path
```
PyTorch (QAT, fake-quant) → ONNX (opset 17) → ORT INT8 static quantization
→ benchmark via onnxruntime with 1 thread
```

### 8.2 Streaming inference
Chunked input of 160 ms (1600 samples) with 40 ms overlap. Maintain PCEN smoothing state across chunks (this is the non-obvious part — PCEN has a recursive smoother; must export it as a stateful op or manage state in host code).

### 8.3 Expected budget on Pi 4B
| Stage            | Time (ms) |
|------------------|-----------|
| MelSpec + PCEN   | ~8        |
| BC-ResNet INT8   | ~35       |
| Both heads       | ~3        |
| Cosine + thresh  | ~1        |
| **Total / chunk**| **~47**   |
xRT = 47/160 ≈ 0.29 — **over budget.** Mitigations in order of preference: reduce base channels 32→24 (saves ~35% compute, ~2% TA), move to 200 ms chunks, or fall back to Cortex-A76-class silicon. Decision due after Stage 1.

## 9. Hyperparameter Search (Optuna, revised)

**Critical change:** search runs on Stage 2 only, not end-to-end. End-to-end Optuna on a 36-hr pipeline is infeasible on Kaggle.

* Sampler: TPE, 40 trials, median pruner.
* Search space: λ weights (log-U[0.1, 10]), T ∈ [2, 5], AAM margin [0.15, 0.35], AAM scale {20, 30, 40}, LR ∈ log-U[5e-5, 5e-3].
* **Objective:** `TA_noisy - 10 · max(0, FA_per_hr - 1)`. Linear penalty, not "heavy" — heavy penalties collapse TPE's acquisition surface.

## 10. Kaggle Execution (sharpened from 1.MD)

* Mount all datasets via `/kaggle/input/` — Kaggle Datasets pre-upload of VoxCeleb1 avoids the broken `torchaudio` download path.
* Checkpoint every epoch **and** every 90 minutes (wall-clock), keyed by Optuna trial ID.
* On resume: reload `model + optimizer + scheduler + Optuna study DB + RNG state`. Omitting RNG state silently changes augmentation seed → non-reproducible resumption.
* `torch.compile(model, mode="reduce-overhead")` after Stage 1 — ~15% train speedup on T4/P100.

## 13. What Changed vs. 1.MD

1. **KD is no longer logit-KL on WavLM** — added a trained probe + hint-layer MSE. This was the biggest correctness bug.
2. **SV embedding dim is 192**, matching ECAPA — removes the silent projection mismatch.
3. **Trunk is partially shared** (split at block 4) — removes KWS↔SV gradient conflict.
4. **PCGrad** added on top of Optuna loss weights.
5. **SNR curriculum** replaces flat -5/30 sampling.
6. **TTS data passes through the real-audio augmentation pipeline** before training.
7. **Hard-negative confusable mining** is now explicit (G2P + phone-Levenshtein), not hand-wavy.
8. **Enrollment + per-user threshold calibration** added — without this, FA<1/hr is unachievable at training time.
9. **Edge deployment path** with a concrete latency budget on declared hardware.
10. **Optuna scoped to Stage 2**, objective function reshaped to linear penalty.
